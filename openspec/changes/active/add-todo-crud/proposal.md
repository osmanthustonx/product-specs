# Proposal: add-todo-crud

## Motivation

產品內的任務與工作事項分散在不同頁面與功能，使用者經常需要「自己在腦中或用其他工具整理」今天要做什麼、先做什麼，導致完成任務的成本偏高。這個需求來自使用者訪談、客服回饋，以及內部團隊實際使用產品時反覆遇到「不知道下一步要做什麼」的情境。我們希望在產品內提供一個集中且結構化的 todo list，提升整體使用體驗與任務完成率。

## Problem Statement

- 使用者無法在同一個地方看到所有待辦事項，只能靠記憶或跳轉多個頁面，容易遺漏重要任務
- 缺乏明確的優先順序與到期提醒，重要任務常被日常雜事淹沒
- 使用者若用其他工具（紙本、備忘錄、第三方 app）整理，增加管理成本，資訊分散也不利團隊協作
- 上述痛點導致使用者在達成目標時感到混亂和挫折

## Expected Outcome

- 使用者能清楚掌握「我現在要做什麼」與「接下來要做什麼」，任務完成率與準時率明顯提升
- 操作路徑縮短，用戶可以少跳好幾個頁面就能完成同一件事
- 產品指標面：提升日／週活躍、功能使用深度與留存率
- 減少因遺漏任務、找不到入口而產生的客服詢問或負面回饋

## Initial Scope

第一版聚焦「個人待辦管理」核心流程：

- 建立與刪除待辦事項
- 編輯內容（標題、描述）
- 設定狀態（未完成／已完成）
- 基本排序（依建立時間或到期日）
- 列表檢視與篩選

後續可擴充：到期時間與提醒、優先級標記、標籤／分類、批次操作、與現有功能（專案、文件、任務板等）關聯同步。
