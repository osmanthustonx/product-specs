# Todo Image Attachment Feature Specification

## Summary
使用者可以為 todo 項目附加單張圖片，在清單頁面以縮圖展示，在詳情頁面以完整尺寸展示。支援上傳、替換、刪除操作。

## Users & Use Cases

### 用戶角色
**一般 Todo 使用者** — 需要透過視覺參考來完成日常任務的使用者。

### 使用情境

| 情境 | 描述 | 預期收益 |
|------|------|--------|
| **購物列表** | 附加商品圖片，快速識別要買什麼 | 減少記憶負擔，加快購物速度 |
| **家務/維修** | 附加現場照片或參考圖，提醒工作細節 | 確保工作品質，避免遺漏 |
| **工作/設計** | 附加設計稿、原型或參考圖 | 團隊溝通更清楚，減少返工 |
| **DIY/手工** | 附加成品圖或步驟照片 | 激勵自己，記錄進度 |

## Functional Requirements

### FR-1: 圖片上傳
- 使用者可在建立或編輯 todo 時，選擇本機檔案上傳圖片
- 支援格式：JPG、PNG、WebP
- 檔案大小限制：5 MB
- 上傳應提供進度反饋

### FR-2: 圖片替換
- 使用者可在編輯 todo 時，替換已附加的圖片
- 流程：移除舊圖片 → 上傳新圖片

### FR-3: 圖片刪除
- 使用者可在編輯 todo 時，移除已附加的圖片
- 刪除操作應提供確認

### FR-4: 清單頁面展示
- 有圖片的 todo 在清單頁面顯示圖片縮圖
- 縮圖尺寸：約 40x40 px（或根據 UI 設計調整）
- 若無圖片，不顯示圖片區域

### FR-5: 詳情頁面展示
- 詳情頁面顯示完整尺寸圖片
- 圖片應適應容器寬度，自動調整高度（maintain aspect ratio）
- 點擊圖片可放大至全屏查看（可選增強）

### FR-6: 圖片儲存和檢索
- 圖片應持久儲存在系統中
- 刪除 todo 時，對應的圖片也應被清除

## Behavioral Spec

### Scenario 1: 新增 todo 並上傳圖片 (Happy Path)
```
Given 使用者在 todo 建立頁面
When 使用者填入 todo 標題
And 使用者點擊「新增圖片」按鈕
And 使用者選擇本機 JPG 檔案（3 MB）
And 系統顯示上傳進度
And 上傳完成後顯示圖片預覽
And 使用者點擊「建立」按鈕
Then 新 todo 應被建立
And 圖片應被附加到該 todo
And 清單頁面應顯示該 todo 的圖片縮圖
```

### Scenario 2: 查看 todo 詳情頁面上的圖片 (Happy Path)
```
Given 一個附有圖片的 todo 存在
When 使用者點擊該 todo
Then 詳情頁面應打開
And 圖片應以完整尺寸顯示
And 圖片下方應顯示編輯/刪除選項
```

### Scenario 3: 替換已有圖片 (Happy Path)
```
Given 一個已附加圖片的 todo
When 使用者進入編輯模式
And 使用者點擊圖片上的「更換圖片」按鈕
And 使用者選擇新的 PNG 檔案
And 系統完成上傳
Then 舊圖片應被替換
And 新圖片應在清單和詳情頁面顯示
```

### Scenario 4: 上傳檔案超過大小限制 (Error Case)
```
Given 使用者在上傳圖片
When 使用者選擇 8 MB 的大檔案
And 系統驗證檔案大小
Then 應顯示錯誤訊息：「檔案過大，請選擇 5 MB 以內的圖片」
And 上傳應被中止
And Todo 應保持未變更狀態
```

### Scenario 5: 上傳不支援的檔案格式 (Error Case)
```
Given 使用者在上傳圖片
When 使用者選擇 GIF 檔案
And 系統驗證檔案格式
Then 應顯示錯誤訊息：「不支援此檔案格式，請選擇 JPG、PNG 或 WebP」
And 上傳應被中止
```

### Scenario 6: 刪除圖片確認流程 (Validation)
```
Given 一個附有圖片的 todo 在編輯狀態
When 使用者點擊「刪除圖片」按鈕
Then 應出現確認對話框：「確定要刪除圖片嗎？」
When 使用者點擊「確定」
Then 圖片應被立即刪除
And 清單和詳情頁面應不再顯示圖片
When 使用者點擊「取消」
Then 圖片應保持不變
```

## Scope

### In Scope ✓
- 單個 todo 可附加 1 張圖片
- 支援 JPG、PNG、WebP 格式
- 檔案大小限制 5 MB
- 清單頁面縮圖展示
- 詳情頁面完整圖片展示
- 上傳/替換/刪除圖片操作
- 圖片驗證（格式、大小）
- 圖片持久儲存與清除

### Out of Scope ✗
- 圖片編輯功能（裁切、濾鏡、旋轉等）
- 支援多張圖片
- 圖片標籤或備註
- 圖片分享功能
- AI 圖片辨識或標籤生成
- 圖片壓縮優化（由客戶端決定）
- 離線圖片存取

## Dependencies & Open Questions

### 技術依賴
- [ ] 影像儲存服務（S3、CDN 或本地儲存？）
- [ ] 影像處理/驗證庫的選擇
- [ ] 前端圖片預覽元件

### 開放問題
1. **圖片儲存位置**: 伺服器儲存 vs 雲端存儲？有無 CDN？
2. **自動縮圖生成**: 伺服器端自動生成縮圖，還是前端展示原圖（縮放顯示）？
3. **圖片上傳進度**: 是否需要上傳進度條？大檔案是否分片上傳？
4. **圖片全屏查看**: 詳情頁面點擊圖片是否支援全屏放大？
5. **圖片加載失敗**: 網絡錯誤時的重試機制？

### 未來擴展可能
- 支援多張圖片（相冊）
- 圖片壓縮最佳化
- 離線模式圖片存取
- 手機拍照上傳
