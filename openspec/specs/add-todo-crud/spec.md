# 個人待辦清單 CRUD

## 1. Summary

提供使用者一個集中的個人待辦清單，支援基本 CRUD 操作與排序／篩選，協助管理每日工作事項。

## 2. Users & Use Cases

**主要使用者角色**：
- 一般使用者（個人任務管理者）
- 進階使用者／團隊成員（需要在產品內追蹤自己相關任務）

**關鍵使用情境**：
- 使用者在操作產品時，臨時想到一件事需要稍後完成，想快速新增一筆待辦事項
- 使用者開始一天工作前，想在同一個地方看到今天所有待辦並排定順序
- 使用者完成某項任務後，想立即勾選完成以保持清單最新
- 使用者想回顧過去幾天已完成的事項，確認進度或作為紀錄
- 使用者在任務變更時，想編輯待辦的標題／描述或刪除不再需要的項目

## 3. Functional Requirements

- 身為使用者，我可以新增一個 todo，並填入標題（必填，最多 200 字元）與描述（選填），以記錄我要做的事情
- 身為使用者，我可以在清單中看到所有尚未完成的 todo，以便掌握目前待辦事項
- 身為使用者，我可以點擊某一筆 todo 進行編輯，修改標題（最多 200 字元）或描述，讓內容保持正確
- 身為使用者，我可以將一筆 todo 標記為已完成，讓清單只顯示需要處理的事項
- 身為使用者，我可以刪除不再需要的 todo，避免清單被無用資訊佔據
- 身為使用者，我可以依建立時間或到期日（若有）排序 todo，方便安排優先順序
- 身為使用者，我可以透過關鍵字搜尋 todo，快速找到特定事項

## 4. Behavioral Spec

### Create Todo

#### Scenario: 新增待辦成功
  Given 使用者已登入並開啟個人 todo 清單頁面
  And 清單上方有「新增待辦」輸入區
  When 使用者在標題欄位輸入有效文字並點擊「新增」
  Then 系統建立一筆新的 todo 並顯示在清單最上方
  And 該 todo 預設狀態為「未完成」

#### Scenario: 新增待辦時標題為空
  Given 使用者已開啟個人 todo 清單頁面
  When 使用者在標題欄位未輸入任何文字就點擊「新增」
  Then 系統阻止建立 todo
  And 在標題欄位下方顯示錯誤訊息「標題為必填欄位」

#### Scenario: 新增待辦時標題超過 200 字元
  Given 使用者已開啟個人 todo 清單頁面
  When 使用者在標題欄位輸入超過 200 字元的文字並點擊「新增」
  Then 系統阻止建立 todo
  And 在標題欄位下方顯示錯誤訊息「標題不可超過 200 字元」

### Edit Todo

#### Scenario: 編輯待辦成功
  Given 清單中已存在至少一筆 todo
  When 使用者點擊其中一筆 todo 的「編輯」
  And 修改標題或描述後點擊「儲存」
  Then 系統更新該 todo 的內容
  And 清單重新顯示更新後的內容

#### Scenario: 編輯待辦時標題超過 200 字元
  Given 使用者已開啟某一筆 todo 的編輯視窗
  When 使用者將標題修改為超過 200 字元並點擊「儲存」
  Then 系統阻止儲存
  And 在標題欄位下方顯示錯誤訊息「標題不可超過 200 字元」

#### Scenario: 編輯待辦儲存失敗（後端錯誤）
  Given 使用者已開啟某一筆 todo 的編輯視窗
  When 使用者修改內容並點擊「儲存」
  And 後端服務發生錯誤（例如 500）
  Then 系統顯示錯誤提示「儲存失敗，請稍後再試」
  And 保留使用者已輸入的內容在編輯視窗中

### Toggle Complete

#### Scenario: 將待辦標記為完成
  Given 清單中有一筆狀態為「未完成」的 todo
  When 使用者勾選該 todo 的完成勾選框
  Then 系統將該 todo 狀態更新為「已完成」
  And 預設從「未完成清單」中隱藏或移至「已完成區塊」

### Delete Todo

#### Scenario: 刪除待辦成功
  Given 清單中有一筆待刪除的 todo
  When 使用者點擊「刪除」並確認
  Then 系統永久刪除該 todo
  And 該 todo 從清單中消失

#### Scenario: 刪除待辦時網路中斷
  Given 清單中有一筆待刪除的 todo
  When 使用者點擊「刪除」並確認
  And 請求送出後網路中斷
  Then 系統顯示錯誤提示「刪除失敗，請檢查網路連線」
  And 該 todo 仍保留在清單中

## 5. Scope

### In Scope
- 個人層級的 todo 清單，支援新增、讀取、更新、刪除
- 基本欄位：標題（必填，最多 200 字元）、描述（選填）、狀態（未完成／已完成）、建立時間
- 基本操作：標記完成／取消完成、刪除、編輯內容
- 基本排序與檢視：依建立時間排序、區分未完成與已完成

### Out of Scope
- 跨使用者共享或指派（例如指派 todo 給其他成員）
- 進階提醒機制（推播、Email 通知等）
- 複雜的權限控管與審批流程
- 高度客製化的分類／標籤管理與報表統計

## 6. Dependencies & Open Questions

### Dependencies
- 使用者帳號與認證系統：需綁定每筆 todo 到特定使用者
- 既有資料庫或儲存服務：需確認是否沿用現有 schema 或建立新資料表
- 通知／排程服務（若之後要支援到期提醒）：需預留介面與事件

### Open Questions
- Todo 是否需要支援「到期日」與「提醒時間」於第一版？若需要，優先級為何？
- 已完成的 todo 要保留多久？是否提供批次清除或自動清除機制？
- 是否需要在行動裝置上提供等同體驗（RWD 或獨立 app 視圖）？
- 是否需要事件紀錄（例如誰在何時建立／修改／刪除），以支援未來審計或除錯？
- Todo 的排序與顯示規則（例如：已完成是否預設折疊、最大顯示筆數）是否需要額外設定頁讓使用者自訂？
